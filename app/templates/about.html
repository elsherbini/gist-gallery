What is this site?

This is a site for sharing code examples. It leverages Github's Gist API. It is intended to be hosted by organizations who want to share code examples both internally and with the public.

How can I get my example on this site?

First, you must be a member of the <%=App.state.org%> organization on Github. If you are a member, simply make a gist at gist.github.com. There are some special files that should be included to make your example awesome:

  • Your gist must have a file named "_include" in order to appear on the site.
  • The information about the gist should be in a "README.md" file.
  • To have a thumbnail on the gallery page, include a "thumbnail.png" file.
  • If your example is a webpage, name the html file "index.html", and it will be rendered in an iframe on the example's page.
  • If your example has some other graphical output, include an "output.png" file to go on the example page.

I'm not a part of the <%=App.state.org%>, but I want this for my organization!

No problem. This app is completely open source, and the code is available on Github at https://github.com/elsherbini/gist-gallery. Information about how to set this up for any public Github organization can be found in the readme of that repository. The included build script generates one HTML file, one CSS, and three JavaScript files, no server-side logic required.

Why did you choose to use Github for code hosting?

Github is quickly becoming the most popular and best featured solution for hosting collaborative coding projects. They recently made huge steps to making it the best place for scientific projects in particular by assigning a DOI to each repository so they can be cited directly, and by giving academic research groups features that normally cost money (read more on their blog here).

How did you come up with such an awesome idea?

This site is inspired by bl.ocks.org by the incredible Mike Bostock, creator of the d3 data visualization library. I wanted a version of bl.ocks.org that didn't require server logic, and that allowed you to display examples at the organization level rather than just the user level. Also I wanted support for examples using languages other than JavaScript. All of those ideas were realized here, but wouldn't have been possible without Mike Bostock's amazing work as a guide.
